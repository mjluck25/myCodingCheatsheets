# Writing a good README.md

Should include:
- Titles and internal titles
  should explain clearly what we have here and should be H1
- Intro (project's aim)
  summary(2-3 sentences)
  if it's a training project, mention your incentive
- Technologies
  languages used, libraries and its versions
- Launch
  local environment, needed data

Consider to add:
- Table of contents
  simple list with the links to headings.
- Illustrations
- Scope of functionalities
- Example of Use
- Project Status
- Sources
- Other info
